当前的python客户端不适合部署在服务器上直接访问，我需要将服务端提供的功能移植为前后端架构，技术栈采用FLASK+streamlit

#### 1. 前端框架安全性部分

尽可能将下面这些安全性部分和其他前端部分在逻辑上分开，便于之后用于其他项目

1. 权限管理的基本单位还是User，每个User保存姓名、手机号、邮箱、微信号、最后登录时间、最后登录IP、角色类别
2. User分为管理员和访客两类，管理员拥有特殊的操作界面，可以管理下面提到的所有功能；而访客只能访问下面明确标注的部分功能
3. 访问网站采用邀请码形式，当输入了正确的邀请码之后为访问者生成一个token用于访问，这里为访问者分配一个访客账户。一共创建10个访客账户
4. 邀请码分为短期、长期、管理员邀请码三类，短期邀请码生成的token有效期为10min，长期邀请码生成的token有效期为1h，管理员邀请码会为访问者分配一个管理员账户
5. token是结合邀请码和访问时间戳生成的JWT令牌，只有持有token才能访问网站，否则重定向到一个提示登录的空界面
6. 登录界面提供管理员登录功能和邀请码输入功能
7. 一个IP一天只能只用一次邀请码
8. 管理员可以对IP设置黑名单，禁止某些IP访问网站
9. 当某个IP登录或输入邀请码频率过高时，直接拒绝该IP的所有访问，将其加入黑名单（100次请求/s）
